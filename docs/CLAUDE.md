## Admonitions (callouts)

Use Docusaurus admonitions for constraints, gotchas, and asides — never inline
bold (`**NOTE:**`), blockquotes, or emoji. Consistent admonition markup lets
both readers and AI agents reliably identify "this is important" and retrieve
it with its context intact.

Use exactly these five types (`:::caution` is deprecated in Docusaurus 3 — do
not use it):

- `:::note` — neutral, supplementary "good to know" context that would clutter
  the main flow.
- `:::tip` — a recommendation, best practice, shortcut, or optional
  enhancement.
- `:::info` — notable contextual facts: version/availability, what a page does
  or doesn't cover, comparisons and definitions.
- `:::warning` — something that can break, surprise, or bite the reader, or
  behavior that is experimental / subject to change. Proceed with care.
- `:::danger` — destructive, irreversible, or security-critical actions (data
  loss, exposing secrets). Use sparingly.

Rules:

1. Give titles in bracket syntax, sentence case: `:::warning[Review before
   sharing]` (capitalize only the first word and proper nouns). The legacy
   space form (`:::warning Review before sharing`) is not used here. Titles are
   optional but improve scannability.
2. One concept per admonition; keep the body concise — the type tag already
   carries the "importance" signal. Don't nest admonitions.
3. Always close with `:::` on its own line.

`hack/gen-command-docs` also emits admonitions; keep it consistent with the
above. The docs lint (`bun run lint`) enforces these rules.
